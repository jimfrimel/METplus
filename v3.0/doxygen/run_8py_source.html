1 """!A shell-like syntax for running serial, MPI and OpenMP programs.
2 
3 This module implements a shell-like syntax for launching MPI and
4 non-MPI programs from Python. It recognizes three types of
5 executables: mpi, "small serial" (safe for running on a batch node)
6 and "big serial" (which should be run via aprun if applicable). There
7 is no difference between "small serial" and "big serial" programs
8 except on certain architectures (like Cray) where the job script runs
9 on a heavily-loaded batch node and has compute nodes assigned for
10 running other programs.
11 
12 @section progtype Program Types
13 
14 There are three types of programs: mpi, serial and "big non-MPI." A
15 "big" executable is one that is either OpenMP, or is a serial program
16 that cannot safely be run on heavily loaded batch nodes. On Cray
17 architecture machines, the job script runs on a heavily-populated
18 "batch" node, with some compute nodes assigned for "large" programs.
19 In such environments, the "big" executables are run on compute nodes
20 and the small ones on the batch node.
21 
22 * mpi('exename') = an executable "exename" that calls MPI_Init and
23  MPI_Finalize exactly once each, in that order.
24 * exe('exename') = a small non-MPI program safe to run on a batch node
25 * bigexe('exename') = a big non-MPI program that must be run on a
26  compute node it may or may not use other forms of parallelism
27 
28 You can also make reusable aliases to avoid having to call those
29 functions over and over (more on that later). Examples:
30 
31 * Python: wrf=mpi('./wrf.exe')
32 * Python: lsl=alias(exe('/bin/ls')['-l'].env(LANG='C',LS_COLORS='never'))
33 
34 Those can then be reused later on as if the code is pasted in, similar
35 to a shell alias.
36 
37 @section serexs Serial Execution Syntax
38 
39 Select your serial programs by exe('name') for small serial programs
40 and bigexe('name') for big serial programs. The return value of those
41 functions can then be used with a shell-like syntax to specify
42 redirection and piping. Example:
43 
44 * shell version: ls -l / | wc -l
45 * Python version: run(exe('ls')['-l','/'] | exe('wc')['-l'])
46 
47 Redirection syntax similar to the shell (< > and << operators):
48 @code
49  run( ( exe('myprogram')['arg1','arg2','...'] < 'infile' ) > 'outfile')
50 @endcode
51 
52 Note the extra set of parentheses: you cannot do "exe('prog') < infile
53 > outfile" because of the order of precedence of Python operators
54 
55 Append also works:
56 @code
57  run(exe('myprogram')['arg1','arg2','...'] >> 'appendfile')
58 @endcode
59 
60 You can also send strings as input with <<
61 @code
62  run(exe('myprogram')['arg1','arg2','...'] << 'some input string')
63 @endcode
64 
65 One difference from shells is that < and << always modify the
66 beginning of the pipeline:
67 
68 * shell: cat < infile | wc -l
69 * Python #1: ( exe('cat') < 'infile' ) | exe('wc')['-l']
70 * Python #2: exe('cat') | ( exe('wc')['-l'] < 'infile' )
71 
72 Note that the last second one, equivalent to `cat|wc -l<infile`, would
73 NOT work in a shell since you would be giving wc -l two inputs.
74 
75 @section parexs Parallel Execution Syntax
76 
77 Use mpi('exename') to select your executable, use [] to set arguments,
78 use multiplication to set the number of ranks and use addition to
79 combine different executables together into a multiple program
80 multiple data (MPMD) MPI program.
81 
82 Run ten copies of ls -l:
83 @code
84  run(mpirun(mpiserial(('ls')['-l'])*10))
85 @endcode
86 
87 Run HyCOM coupled HWRF: one wm3c.exe, 30 hycom.exe and 204 wrf.exe:
88 @code
89  run(mpirun(mpi('wm3c.exe') + mpi('hycom.exe')*30 + mpi('wrf.exe')*204))
90 @endcode
91 
92 You can set environment variables, pipe MPI output and handle
93 redirection using the mpirun() function, which converts MPI programs
94 into an bigexe()-style object (Runner):
95 
96 Shell version:
97 @code{.unformatted}
98  result=$( mpirun -n 30 hostname | sort -u | wc -l )
99 @endcode
100 
101 Python version:
102 @code
103  result=runstr( mpirun(mpi('hostname')*30) | exe['sort']['-u'] | exe['wc']['-l'] )
104 @endcode
105 
106 @section aliases Aliases
107 
108 If you find yourself frequently needing the same command, or you need
109 to store a command for multiple uses, then then you should define an
110 alias. Let's say you want "long output" format Japanese language "ls"
111 output:
112 
113 @code
114  exe('ls')['-l','/path/to/dir'].env(LANG='JP')
115 @endcode
116 
117 but you find yourself running that on many different directories.
118 Then you may want to make an alias:
119 
120 @code
121  jplsl=alias(exe('ls')['-l'].env(LANG='JP'))
122 @endcode
123 
124 The return value jplsl can be treated as an exe()-like return value
125 since it was from exe() originally, but any new arguments will be
126 appended to the original set:
127 
128 @code
129  run(jplsl['/path/to/dir'])
130 @endcode
131 
132 Note that if we did this:
133 @code
134  badlsl=exe('ls')['-l'].env(LANG='JP') # Bad! No alias!
135  run(badlsl['/']) # will list /
136  run(badlsl['/home']) # will list / and /home
137  run(badlsl['/usr/bin']) # will list / /home and /usr/bin
138 
139  goodlsl=alias(exe('ls')['-l'].env(LANG='JP')
140  run(goodlsl['/']) # will list /
141  run(goodlsl['/home']) # will list /home
142  run(goodlsl['/usr/bin']) # will list /usr/bin
143 @endcode
144 
145 Then the run(badlsl['/home']) would list /home AND / which is NOT what
146 we want. Why does it do that? It is because badlsl is not an alias
147 --- it is a regular output from exe(), so every time we call its []
148 operator, we add an argument to the original command. When we call
149 alias() it returns a copy-on-write version (goodlsl), where every call
150 to [] creates a new object.
151 
152 Note that alias() also works with pipelines, but most operations will
153 only modify the last the command in the pipeline (or the first, for
154 operations that change stdin).

183 def make_mpi(mpi_name=produtil.mpi_impl.NO_NAME,**kwargs):
184  """!Creates an MPI implementation object for the specified MPI
185  implementation.
186 
187  Creates an object suitable for passing to the mpiimpl argument of
188  various functions in produtil.run. The object will implement the
189  requested MPI implementation. This object is NOT used to
190  initialize the produtil.run; instead it is simply returned.
191 
192  @returns an MPI implementation object for the specified arguments
193 
194  @param mpi_name the name of the MPI implementation. If
195  mpi_name=None, an mpiimpl for pure serial programs is returned.
196  If no mpi_name is given, the local machine's available MPI
197  implementation is detected and returned.
198 
199  @note When mpi_name is unset, the detection of the MPI
200  implementation is redone, even if it had been done before by
201  another call to get_mpi() or detect_mpi()
202 
203  @param kwargs additional arguments passed to produtil.mpi_impl.get_mpi()
204 
205  @see produtil.mpi_impl.impi.Implementation.detect()"""
206 
207  # The special value of None requests no MPI implementation. That
208  # situation does not need any initialization of the produtil.run
209  # module:
210  if mpi_name is None:
211  return produtil.mpi_impl.get_mpi(None,**kwargs)
212

214  # produtil.run is initialized to something before running
215  # produtil.mpi_impl.get_mpi.

221 def detect_mpi():
222  """!Called by functions inside produtil.run to automatically
223  detect the available MPI implementation.
224 
225  Detects the available MPI implementation - but that isn't as
226  simple as it sounds. The detection logic requires calling
227  produtil.run. Hence, produtil.run must be usable during the call
228  to detect_mpi(). The way this is handled is in three steps:
229 
230  1. Set the MPI implementation to "no MPI." This will allow all
231  serial (non-OpenMP, non-MPI) calls to succeed.
232 
233  2. Run the MPI implementation detection functions. These will be
234  able to use produtil.run to execute serial programs to detect
235  the MPI implementation.
236 
237  3. If an MPI implementation is detected, set the MPI
238  implementation to that one.
239 
240  @note MPI implementation detection only happens during the first
241  call to detect_mpi(). Any later calls will return the cached
242  result from _detected_mpi.
243 
244  @returns a subclass of
245  produtil.mpi_impl.mpi_impl_base.ImplementationBase for generation
246  of produtil.prog.Runner objects from openmp or mpi program
247  specifications.
248 
249  @see _detected_mpi"""
250  global _detected_mpi
251  if _detected_mpi: return _detected_mpi
252 
253  # First, set the implementation to None so that the mpi_impl
254  # subclasses can use produtil.run. This will only allow serial,
255  # non-OpenMP programs:
256  _detected_mpi=produtil.mpi_impl.get_mpi(None)
257 
258  # Next, ask produtil.mpi_impl to detect the MPI implementation:
259  detected=produtil.mpi_impl.get_mpi()
260 
261  # If detection succeeds, override the selected MPI implementation:
262  if detected:
263  _detected_mpi=detected
264 
265  return _detected_mpi

327 def make_pipeline(arg,capture,mpiimpl=None,**kwargs):
328  """!This internal implementation function generates a
329  prog.PopenCommand object for the specified input, which may be a
330  prog.Runner or mpiprog.MPIRanksBase.
331  @param arg the produtil.prog.Runner to convert. This is the
332  output of exe(), bigexe() or mpirun()
333  @param capture if True, capture the stdout into a string
334  @param kwargs additional keyword arguments, same as for mpirun()"""
335  if mpiimpl is None:
336  mpiimpl=detect_mpi()
337  if isinstance(arg,prog.Runner):
338  runner=arg
339  elif isinstance(arg, mpiprog.MPIRanksBase):
340  runner=mpiimpl.mpirunner(arg,**kwargs)
341  else:
342  raise InvalidRunArgument(
343  'Can only run a Runner object (such as from exe()) or an '
344  'MPIRanksBase object (such as from mpi() or mpiserial()). '
345  'Got: %s'%(repr(arg),))
346  logger=None
347  if 'logger' in kwargs: logger=kwargs['logger']
348  if logger is not None:
349  logger.info('Starting: %s'%(repr(arg),))
350  if capture: logger.info(' - and will capture output.')
351  b=kwargs.get('binary',False)
352  b=bool(b)
353  pl=pipeline.Pipeline(runner,capture=capture,logger=logger,
354  binary=b)
355  if logger is not None:
356  logger.debug('Pipeline is %s'%(repr(pl),))
357  return pl

380 def waitprocs(procs,logger=None,timeout=None,usleep=1000):
381  """!Not implemented: background process monitoring
382 
383  Waits for one or more backgrounded processes to complete. Logs to
384  the specified logger while doing so. If a timeout is specified,
385  returns False after the given time if some processes have not
386  returned. The usleep argument is the number of microseconds to
387  sleep between checks (can be a fraction). The first argument,
388  procs specifies the processes to check. It must be a
389  produtil.prog.Pipeline (return value from runbg) or an iterable
390  (list or tuple) of such.
391 
392  @bug produtil.run.waitprocs() is untested
393 
394  @warning This is not tested and probably does not work.
395 
396  @param procs the processes to watch
397  @param logger the logging.Logger for log messages
398  @param timeout how long to wait before giving up
399  @param usleep sleep time between checks"""
400  p=set()
401  if isinstance(procs,produtil.prog.PopenCommand):
402  p.add(procs)
403  else:
404  for pp in procs:
405  p.add(pp)
406  if logger is not None: logger.info("Wait for: %s",repr(p))
407  while p: # keep looping as long as there are unfinished processes
408  p2=set()
409  for proc in p:
410  ret=proc.poll()
411  if ret is not None:
412  if logger is not None:
413  logger.info("%s returned %s"%(repr(proc),repr(ret)))
414  elif logger is not None and usleep>4.99999e6:
415  # babble about running processes if the sleep time is long.
416  logger.info("%s is still running"%(repr(proc),))
417  p2.add(proc)
418  p=p2
419 
420  if not p: break # done! no need to sleep...
421 
422  if usleep>4.99999e6 and logger is not None:
423  # babble about sleeping if the sleep time is 5sec or longer:
424  logger.info("... sleep %f ..."%(float(usleep/1.e6),))
425  time.sleep(usleep/1.e6)
426  return False if(p) else True

434 def run(arg,logger=None,sleeptime=None,**kwargs):
435  """!Executes the specified program and attempts to return its exit
436  status. In the case of a pipeline, the highest exit status seen
437  is returned. For MPI programs, exit statuses are unreliable and
438  generally implementation-dependent, but it is usually safe to
439  assume that a program that runs MPI_Finalize() and exits normally
440  will return 0, and anything that runs MPI_Abort(MPI_COMM_WORLD)
441  will return non-zero. Programs that exit due to a signal will
442  return statuses >255 and can be interpreted with WTERMSIG,
443  WIFSIGNALLED, etc.
444  @param arg the produtil.prog.Runner to execute (output of
445  exe(), bigexe() or mpirun()
446  @param logger a logging.Logger to log messages
447  @param sleeptime time to sleep between checks of child process
448  @param kwargs ignored"""
449  p=make_pipeline(arg,False,logger=logger)
450  p.communicate(sleeptime=sleeptime)
451  result=p.poll()
452  if logger is not None:
453  logger.info(' - exit status %d'%(int(result),))
454  return result

494 def runstr(arg,logger=None,**kwargs):
495  """!Executes the specified program or pipeline, capturing its
496  stdout and returning that as a string.
497 
498  If the exit status is non-zero, then NonZeroExit is thrown.
499 
500  Example:
501  @code
502  runstr(exe('false'),ret=(1))
503  @endcode
504 
505  succeeds if "false" returns 1, and raises ExitStatusError otherwise.
506 
507  @param arg The "arg" argument must be from mpiserial, mpi, exe or
508  bigexe.
509  @param logger a logging.Logger for logging messages
510  @param kwargs You can specify an optional list or tuple "ret" that

512  codes are zero or positive: negative values represent

516  s=p.to_string()

519  if not r in kwargs['ret']:
520  raise ExitStatusException('%s: unexpected exit status'%(repr(arg),),r)
521  elif not r==0:
522  raise ExitStatusException('%s: non-zero exit status'%(repr(arg),),r)
