Detailed Description

Represents the context from which a BaseObject is accessed.

The Context is used during any evaluation of a BaseObject to a logical, numeric, block of bash code, or other type. A Context consists of the relevant token (for filename and line number), the run mode (baseline or execution), the call stack, and logging information. The BaseObject subclasses use that information for variable resolution or error reporting.
